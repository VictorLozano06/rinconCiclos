# Apuntes Rastrollo

## Angular y RxJS

### standalone
- Un componente `standalone` no necesita declararse dentro de un modulo clasico.
- Lleva sus propios `imports` dentro del decorador `@Component`.
- Sirve para que el componente sea mas independiente y facil de reutilizar.

### Observable
- Un `Observable` es una fuente de datos que puede emitir valores ahora o mas tarde.
- Angular lo usa mucho en rutas y en HTTP.
- Un servicio puede devolver un `Observable<RecursoDto[]>` aunque todavia no tenga los datos cargados.

Ejemplo:

```ts
getTodos(): Observable<RecursoDto[]> {
  return this.http.get<RecursoDto[]>('/api/recursos');
}
```

- Aqui `getTodos()` no devuelve ya el array final.
- Devuelve un `Observable` que emitira ese array cuando la peticion termine.

### subscribe
- `subscribe` sirve para escuchar un `Observable`.
- Hasta que haces `subscribe`, normalmente no consumes el resultado.
- En rutas significa "cada vez que cambie esto, ejecuta esta funcion".
- En HTTP significa "cuando llegue la respuesta, haz esto".

### Diferencia entre Observable y subscribe
- `Observable`: el canal o flujo de datos.
- `subscribe`: la accion de engancharte a ese flujo para recibir lo que emite.

Ejemplo:

```ts
const flujo = this.recursoService.getTodos(); // Observable

flujo.subscribe((recursos) => {
  console.log(recursos); // aqui consumes el dato
});
```

### callback
- Un `callback` es una funcion que se le pasa a otra funcion para que la ejecute despues.
- Ejemplo sencillo:

```ts
this.route.params.subscribe((params) => {
  console.log(params);
});
```

- Lo que va dentro de `subscribe(...)` es un callback.
- O sea: "cuando llegue el dato, ejecuta esta funcion".

### snapshot
- `snapshot` es una foto fija del estado actual de la ruta.
- No escucha cambios futuros.
- Solo lee el valor que existe en ese momento.

### forkJoin
- `forkJoin` lanza varias peticiones en paralelo.
- Espera a que todas terminen.
- Cuando acaban todas, entra en `next`.
- Si una falla, falla el bloque entero.

Ejemplo:

```ts
forkJoin({
  categorias: this.categoriaService.getCategorias(),
  recurso: this.recursoService.getDetalle(idCategoria, numRecurso)
}).subscribe(({ categorias, recurso }) => {
  // aqui ya tienes ambas respuestas
});
```

## Bucles

### for...of
- Recorre elementos de un array.
- similar a foreach
- No espera eventos ni datos async.

Ejemplo:

```ts
for (const recurso of recursos) {
  for (const ciclo of recurso.ciclos || []) {
    console.log(ciclo.nombre);
  }
}
```

### foreach
- `forEach` tambien recorre arrays, pero con callback.
- Ejemplo:

```ts
recursos.forEach((recurso) => {
  console.log(recurso.nombre);
});
```

## Ruta de navegacion superior

- Es la linea de texto que suele salir arriba para ubicarte.
- Ejemplo:
  `Inicio / Evaluaciones / 1ª Evaluacion / Acta final`
- En muchos proyectos se llama `breadcrumbs`, pero Angular no te lo da hecho.
- Es solo un nombre que el equipo elige para representar esa ruta.
- En este proyecto lo dejamos con un nombre mas claro: `rutaNavegacion`.

## DTO

- Un DTO es un objeto con forma de datos que viene del backend o que se manda al backend.
- Ejemplo: `RecursoDto`.
- Sirve para tipar y entender que campos trae cada respuesta.

## ActivatedRoute

- Es el servicio de Angular que te deja leer la ruta actual.
- Con `params.subscribe(...)` escuchas cambios de parametros.
- Con `snapshot` lees el valor actual sin escuchar cambios.
